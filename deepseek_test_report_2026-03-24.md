# DeepSeek Prompt Test Report (2026-03-24)

## 测试环境

- 模型：`deepseek-chat`
- 方式：最小行为测试 + 真实场景补充测试 + 修正后复测
- 目标：验证健身/穿搭分支是否遵守 AKM 逻辑，即先建模、再裁决

## 测试项

1. Fitness elicitation
2. Fitness execution
3. Fashion elicitation
4. Fashion execution
5. Fitness execution retest after prompt tightening

## 结果摘要

### Fitness elicitation

结论：**通过**

观察：
- 能先收主线目标、身体限制、器械环境、时间预算
- 没有直接越过建模给完整训练计划
- 能显式列出 `MissingInputs`
- 在真实场景测试中会先逼用户给目标排序和伤病细节，没有把“想瘦也想拳击表现好”直接揉成一个目标

问题：
- 更像结构化缺口罗列，而不是强势追问
- 英文输出时偶有解释性扩写，但不影响方法判断

### Fitness execution

结论：**基本通过，但首次实测暴露出默认分化训练脑补**

观察：
- 能遵守约束优先
- 能给出结构化裁决
- 能显式暴露 `DecisionConfidence`
- 会提示恢复状态、肩背状态、目标排序等缺失信息

问题：
- 在首次真实场景测试里，模型仍会基于“昨练上肢”弱证据滑向“今天练下肢”的常见分化假设
- 这说明“不得脑补训练顺序”还需要更硬的拒答规则，而不只是提醒降低置信度

修正：
- 已追加“最近训练顺序不足时，不得默认上肢后接下肢”
- 已追加“当疼痛、恢复、目标排序三者中任意两项缺失时，不得输出完整训练单”
- 已允许 `PrimaryDecision` 直接返回“先补关键信息，再决定今天练什么”

### Fitness execution retest after prompt tightening

结论：**通过，已明显改善**

观察：
- 复测时不再把“昨练上肢”自动翻译成“今练下肢”
- 会先明确说明当前无法判定最优训练刺激
- `PrimaryDecision` 已能退回到“defer full programming”
- 输出转为低风险占位动作与缺失信息，而不是伪装成完整计划

残留问题：
- 即使已经低置信度拒绝完整处方，模型仍倾向给一个简短的保守活动方案
- 这对真实使用未必是坏事，但若后续要做更强硬的 fail-fast 版本，可以继续把低置信度条件下的输出压缩为“状态筛查 + 待确认项”，减少动作建议

### Fashion elicitation

结论：**通过**

观察：
- 能先问场景、体型、风格、衣橱、功能约束
- 没有在输入不足时直接给搭配方案
- 缺口识别准确
- 在真实场景测试中，能继续往场景正式度、外套类型、骑行细节、体型优先级上追问，而不是急着推荐单品

问题：
- 当前风格偏好挖掘仍可更锋利，但结构方向正确
- 输出略长，后续若用于商店页展示，最好再压缩为更短的追问风格

### Fashion execution

结论：**通过**

观察：
- 能根据场景、体型、衣橱与功能约束输出裁决
- 没有明显把风格标签直接当方案
- 能给出采购优先级
- 能把“会议 + 电助力通勤”的双场景冲突收束成单套方案，而不是拆成两套互不兼容的建议

问题：
- 仍会对现有衣橱覆盖度做一定推断，但已被 `MissingInputs` 补救
- 会自然补充一些审美解释语句，适合公开版，不一定适合最短执行版

## 总判断

- Fitness：已具备可发布 skill 的行为基础；执行 prompt 已从“默认脑补训练分化”收紧为“先承认信息不足，再给低风险占位建议”
- Fashion：已具备可发布 skill 的行为基础；更像系统设计型 skill，行为稳定度已经够用
- 两条线都已证明：这不是单 Prompt，而是 AKM 的场景分支方法

## 对论文的意义

- Fitness 分支可写成 **n=1 长期纵向自用系统**，并可加入“外部模型行为测试”作为方法边界验证，而非疗效验证
- Fashion 分支可写成 **系统设计与方法论文**，并可加入“外部模型在衣橱约束下未退化为审美套话”的行为证据
