# AKM 即刻 / Telegram 自动推广方案

## 目标定位

这不是大规模运营方案。
这是一轮 **低社交负载、全自动发出** 的连续输出试验。

核心目标只有四个：

1. 口径稳定
2. 节奏稳定
3. 不失忆
4. 人工介入可控

不追求：

- 涨粉
- 爆量
- 互动率
- 自动回复评论
- 自动寻找热点

## 平台选择

第一阶段只跑两个平台：

- 即刻
- Telegram Channel

不纳入第一阶段：

- X / Twitter
- Discord
- 小红书
- 视频平台

原因很简单：

- 即刻适合中文文本连续输出
- Telegram 自动化摩擦最低
- 两个平台能同时覆盖“中文公开文本试验”与“低风控自动频道”

## 试验周期

第一阶段固定 14 天。

14 天内不因为单条数据波动改变策略。
只观察：

- 口径是否漂移
- 主题是否乱跳
- 输出是否重复
- 是否频繁需要人工修正

## 内容主线

只允许围绕三条主线发：

1. AKM
   - profile-first
   - persistent context surfaces
   - 为什么平台有承载面但没有填法
2. DaE
   - upstream profiling
   - generic-answer failure
   - 顾问协作为什么先做人设/状态建模
3. 分支示例
   - fitness
   - fashion
   - 样本记录能说明什么

禁止在第一阶段主动扩出新主线。

## 发帖节奏

### 即刻

- 每日 1 条
- 以短帖为主
- 每条 120-300 字为宜
- 不追求线程式长文

### Telegram

- 每日 1 条或隔日 1 条
- 可比即刻略长
- 以完整观点卡片为主
- 每条 150-500 字为宜

## 平台适配规则

同一主题池，同一立场，不同平台只允许轻微重写。

允许变化：

- 标题语气
- 句长
- 开头钩子
- 行文密度

不允许变化：

- 方法主张
- 关键术语
- 立场方向
- 对外链接主链

## 内容结构模板

每条内容默认采用四段：

1. 一个问题
2. 一个判断
3. 一个例子或分支锚点
4. 一个公开链接

链接优先级：

1. AKM 母港
2. DaE 分支
3. Fitness / Fashion 分支
4. SSRN 论文
5. ClawHub skill

## 自动化工作流

建议工作流固定为：

1. 从主题池抽一条当日主题
2. 生成即刻版和 Telegram 版两份文案
3. 规则检查
   - 是否偏离 AKM 主线
   - 是否出现新术语污染
   - 是否出现过度营销语气
   - 是否重复近 7 天文案
4. 定时发出
5. 记录日志

## 人工介入边界

允许人工做的事：

- 每日一次抽查
- 删除明显跑偏内容
- 修正链接错误
- 暂停自动发布

不允许人工做的事：

- 每条都重写
- 临时追热点
- 为数据焦虑而频繁改策略

人工介入预算：

- 日均不超过 30 分钟

超过这个值，说明当前自动化方案不合格。

## 失败条件

出现以下任一情况，应立即暂停试验并复盘：

- 连续 3 天明显偏离 AKM / DaE / 分支主线
- 高频重复表达同一判断
- 术语漂移，开始自造新概念
- 人工介入显著超时
- 平台风控或账号异常

## 成功标准

14 天后，如果满足以下条件，可进入第二阶段：

- 主题与立场稳定
- 文本质量没有明显衰减
- 两平台没有严重风控问题
- 人工介入低于预算
- 至少能把 GitHub / SSRN / ClawHub 三端资产稳定串起来

## 第二阶段决策门槛

14 天不是决定“是否成功商业化”。
只是决定一件事：

**这条线值不值得继续跑满 1 个月，并在之后讨论是否收编为 AKM 正式分支。**

在跑满 1 个月之前，不把它写进 AKM 母港正式 branch。
